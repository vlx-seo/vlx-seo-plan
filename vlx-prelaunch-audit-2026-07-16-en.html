Methodology & scope

Audit run live on 2026-07-16 with an in-browser fetch() crawler over the 184 sitemap URLs plus targeted probes. Both environments sit behind Cognito, which shapes what can and cannot be measured.

🔐
Access — dev = prod
The team confirmed dev.vlx.ai and prod.vlx.ai serve the same build. prod.vlx.ai blocked automated access, so the live verification was done on dev.vlx.ai and applies equally to prod. Both are gated by AWS ALB + Amazon Cognito (authenticated session required).
🚫
External tools blocked by Cognito
PageSpeed Insights, Lighthouse, CrUX and any public crawler cannot reach the site — they don't hold the Cognito session cookie. All performance data comes from in-browser Performance APIs (lab, not field). Real field LCP/INP can only be measured with PageSpeed once Cognito is removed (post-launch).
🧰
Backlink / keyword tooling unavailable this session
Backlinks, keyword rankings, SERP and ASO could not be pulled live and are marked PROVISIONAL (they don't depend on the site code and were unchanged since the June audit).
📐
Client rule — indexing in pre-launch environments
Per the client (2026-07-16): pre-launch environments should serve noindex, nofollow. See the dedicated Indexing status section for the current state and the nuance.

Even though the meta robots configuration is wrong, the Amazon Cognito wall blocks search engines entirely, so no search engine is indexing the site right now. There is no live SEO damage — the current state is correct for a pre-launch environment.

Before the site is exposed, ensure all public pages have index, follow correctly set and that this is driven by the environment (pre-prod → noindex,nofollow; production → index,follow). Priority focus

Load speed

Infrastructure is fast when warm (TTFB ~85 ms, CLS 0, optimized images). The load-speed lever is the heavy JavaScript bundle plus a large SSR DOM — under real mobile CPU throttling this delays FCP and therefore LCP. Images are not the problem.

📊
Historical context & recommendation (P1)
The last real throttled measurement (PageSpeed, June, hub) was LCP ≈ 7.6 s, diagnosed as FCP/render-path limited — not the image. Current resource weights confirm the diagnosis persists. Fix: code-split and defer below-the-fold JS, purge unused JS/CSS, lighten the home page, then re-measure mobile LCP <2.5 s with PageSpeed once the site is public.

Post-launch validation checklist

Run in order as soon as Cognito is removed and the site is public on vlx.ai.

A · Indexing & crawling (first hours)

  • meta robots = index, follow on all public pages (the opposite of pre-launch).
  • Confirm /blog/ and /contact/ are now index,follow — the pre-prod noindex did not propagate.
  • robots.txt correct; AI posture intact; Sitemap: → https://vlx.ai/sitemap.xml.
  • sitemap.xml returns 200 with no redirecting URLs (clean up the 8) and no 404s.
  • Canonicals point to https://vlx.ai/…. Submit sitemap in GSC + request indexing of home and hub. Submit key URLs via IndexNow/Bing.

B · Redirects & migration (day 0)

  • Every vlx.ai (WordPress) URL with traffic/links resolves or redirects in 1 hop, no 404s.
  • Active paid landing pages resolve 200 (not 404).
  • Redirects are 301 (or 308), no A→B→C chains. visualogyx.com still redirects to vlx.ai.

C · Real performance (day 0–1, Cognito removed)

  • PageSpeed mobile on home, hub and top industry page → record LCP/INP/CLS/FCP/TTFB.
  • Confirm hub LCP < 2.5 s (historical ~7.6 s; focus on reducing JS/FCP).
  • Check INP with the real JS bundle; confirm TTFB is stable (no ~900 ms spikes).

D · Analytics & tracking (day 0)

  • GA4 recording pageviews once per navigation (no GTM + direct double-counting).
  • Conversion events (Start Free Trial / Book a Demo) firing. GSC ↔ GA4 linked.

E · Data & monitoring (week 1)

  • GSC coverage with no mass errors; sitemap "success".
  • Export GSC/GA4 baseline to measure migration impact.
  • Field Core Web Vitals (CrUX) populate ~28 days post-launch.
  • Monitor rankings and organic traffic vs the WordPress baseline.

F · Schema & social (week 1)

  • Validate rich results (Organization, SoftwareApplication, FAQ, Article, Breadcrumb) in GSC.
  • og:image ≥ 1200×630 on the 12 industries + key pages; validate in Meta/X/LinkedIn.
