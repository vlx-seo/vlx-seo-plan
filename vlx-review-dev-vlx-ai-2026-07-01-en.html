Methodology note

This pass does NOT repeat the full 12-dimension audit from June 19 — there are real tooling access limitations, and some dimensions don't apply to a pre-launch environment without a verified domain.

🔐
Access to dev.vlx.ai
Protected with AWS ALB + Amazon Cognito (email/password OAuth2 login, not basic auth). The browser already had an active session — the credentials Lau provided (laura.ceballos@softwarecraftcr.com) were not needed.
🚫
External tools blocked
PageSpeed Insights, remote Lighthouse, CrUX, and any public crawler cannot audit this URL — they don't have the Cognito session cookie. Everything about performance in this report comes from the Navigation Timing API measured manually in the authenticated browser, not from real Lighthouse.
📊
GSC / GA4 / Backlinks / ASO / GMB
dev.vlx.ai is not a verified GSC property — no traffic baseline applies yet. Backlinks, keywords, ASO, and GMB don't depend on the site's code and were not re-run in this pass (no change since June 19).
🔴
Open question for Hans/Abe
The homepage H1 now reads "The AI-Native Enterprise Platform for [Inspections/...]" (typewriter-style animation) — a different brand positioning from what we saw on June 19 ("Run Your Inspection Business on VLX"). This could be normal evolution of the Hans-review branch, or a different environment entirely. Confirm which branch is deployed on dev.vlx.ai before treating this comparison as final.

Method: a custom crawler run inside the authenticated browser (fetch() from the page, inheriting the session cookie) across all 185 sitemap URLs, plus targeted checks of legacy WordPress URLs.
